## v0.7.0 (2026-04-04)

### 🏃 跑步运动全面支持 — 活动详情页运动类型感知

#### 活动详情页 (`/activity/{id}`)
- **运动类型自动识别** — 根据 sport 字段自动切换跑步/骑行布局
- **跑步专有指标卡** — 平均配速 (min/km)、最快配速、步频 (spm)、VDOT、触地时间 (ms)、垂直振幅 (cm)、步幅 (cm)、垂直步幅比 (%)
- **跑步时间线图表** — 配速曲线（Y轴反转，越低越快）+ 心率 + 步频，替代骑行的功率曲线
- **跑步分段表格** — 每圈显示配速/心率/步频，替代骑行的功率/速度/踏频
- **智能隐藏** — 跑步页面不显示功率区间、PDC 曲线等骑行专有模块
- 骑行页面布局完全不受影响

#### AI 复盘 — 运动类型感知
- **跑步 AI 分析师** — 专用 prompt 关注配速控制、步频一致性、VDOT 趋势、跑步经济性、跑姿指标
- **骑行 AI 分析师** — 保持原有 prompt（功率稳定性、eFTP 变化、功率区间表现等）
- AI 复盘标题自动显示"AI 跑步复盘"或"AI 骑行复盘"

#### FIT 解析器增强 (`engine/fit_parser.py`)
- 新增跑步动态字段解析：stance_time（触地时间）、vertical_oscillation（垂直振幅）、step_length（步幅）、vertical_ratio（垂直步幅比）
- 同时支持逐秒记录和 session 级别平均值

#### 数据库迁移 (`engine/database.py`)
- activities 表新增 4 列：avg_stance_time_ms、avg_vertical_osc_cm、avg_step_length_cm、avg_vertical_ratio
- 自动迁移函数，已有数据库无需手动操作

#### Bug 修复
- **修复 start_time 时区** — 活动开始时间从 UTC 转换为本地时区（北京时间），不再显示凌晨时间
- **修复跑步步频** — FIT 文件存储单腿 strides/min，自动 ×2 转换为总步频 spm

## v0.5.0 (2026-03-18)

### 🎯 商用级训练决策系统重构 — 从"数据展示"升级为"结论→证据→动作"架构

#### 核心改造原则
- 所有页面按"结论层 → 证据层 → 动作层"组织
- 删除低解释性、伪精确、装饰型模块
- 统一颜色语义（蓝=骑行/中性、绿=正向、黄橙=警惕、红=偏离、紫=AI/力量）
- 置信度改为多因子数组，不再是单一原因
- 训练完成率口径分离：主训练（骑/跑）vs 力量

#### 删除的模块
- **肌肉疲劳热力图** — 伪精确、不可审计、易误导（替代为"恢复约束"文字卡）
- **身体评分主卡** — 不可解释、低决策价值（替代为"身体组成状态"结论卡）
- **Body Battery 空卡** — 无数据时不再显示占位
- **合规追踪 0/10** — 抽象总分无意义（替代为"本周约束满足情况"清单）
- **TSB 重复展示** — 同页不再出现 3 次 TSB

#### 新增功能
- **`/api/decision-summary`** — 统一决策对象，面板+计划页共用（避免文案漂移）
- **`/api/constraint-status`** — 本周约束满足情况（休息日/运动频率/连续高负荷/强度日）
- **异常复核规则** — 5 个触发条件（RHR连续偏高/HRV连续下降/睡眠连续不足/用户标记伤病/数据缺失），触发时显示人工复核提示而非强结论
- **confidence_reasons 数组** — 置信度原因改为多因子，不写死单一原因
- **口径分离** — primary_sessions（骑/跑）vs strength_sessions 独立统计

#### 面板页 (`/`)
- 新增 **置信度原因列表** — 展示每条置信度判断依据
- 新增 **异常复核提示** — 触发时显示警告卡片（条件+原因）
- **近7日训练卡增强** — 显示总时长+总TSS+运动分布
- 新增 **页脚免责声明**

#### 身体数据页 (`/body-data`)
- **身体评分 → 身体组成状态卡** — 显示状态标签/判定/关键变化/建议（快读版）
- **趋势结论卡折叠化** — 默认快读版，展开显示三段式详情
- **Body Battery 条件渲染** — 无数据时隐藏
- InBody 区重命名为 **"身体组成明细（数据来源：InBody）"** + 说明文字
- 删除 **身体评分进度条**
- 新增 **页脚免责声明**

#### 训练计划页 (`/plan`)
- **删除肌肉疲劳热力图** + 相关 CSS 清理
- **AI 决策依据默认折叠** — 用 `<details>` 包裹
- **TSB 建议 → 恢复约束卡** — 简洁一段话
- **合规追踪 → 约束满足清单** — ✅/❌/🔄 状态
- **操作区分组** — 主要操作（生成/重新生成/下周）+ 高级操作（降低负荷/复用排布/修改档案）
- 新增 **页脚免责声明**

#### 安全
- 无新增外部接口暴露
- 新 API 均为内部页面调用
- 页脚声明明确"不替代医疗诊断"

## v0.4.0 (2026-03-18)

### 🎯 Decision Cockpit v1 — 从"数据展示"升级为"训练决策"

#### 核心：结论层引擎 (`engine/readiness.py`)
- **训练就绪度评估** (`/api/readiness`) — 基于 HRV/RHR/睡眠/TSB 的矩阵判断，输出今日训练建议（可执行关键课/正常训练/恢复/休息）+ 置信度 + 依据
- **周执行偏差分析** (`/api/weekly-deviation`) — 计划 vs 实际（训练次数/TSS），自动判断正常/略落后/明显落后/过载，给出调整建议
- **身体趋势结论** (`/api/body-trend-summary`) — 基于规则模板自动生成身体组成/恢复状态/训练含义三段式结论
- **指标对比数据** — 所有关键指标提供 vs 7日均值 / vs 上次 / vs 30天 对比

#### 面板页重构 (`/`)
- 新增 **ReadinessCard** — 页面第一屏显示今日训练建议，支持展开查看完整判断依据
- 新增 **MetricCard** 统一组件 — 6 张指标卡（HRV/睡眠/RHR/体能状态/本周TSS/7日训练），每张显示数据时间+来源+对比口径
- 新增 **WeeklyProgressBar** — 本周执行概览（进度条+偏差判断），点击跳转计划页
- **训练负荷图增强** — 时间范围切换（30/90/180天）+ Daily TSS 柱状图叠加 + hover 标记今天
- HRV/睡眠/RHR 卡片点击可跳转身体数据页

#### 身体数据页增强 (`/body-data`)
- 新增 **TrendConclusionCard** — 身体组成结论 + 恢复状态结论 + 训练含义，自动生成
- 体测对比数据增强 — vs 上次 / vs 30天均值 双口径对比

#### 训练计划页增强 (`/plan`)
- 新增 **DeviationPanel** — 训练/TSS 进度条 + 偏差判断 + 跳过训练列表
- 新增 **AI 决策依据卡** — 阶段判定/负荷依据/恢复依据/限制条件，计划不再黑箱
- 新增 **就绪度卡** — 今日训练状态 + 建议，辅助决定是否执行计划

#### Bug 修复
- 修复 HTTP 内网访问无法登录问题 — Cookie `secure` 属性根据请求协议动态判断

#### 安全
- 新增 API 不要求 API key（readiness/deviation/body-trend 为页面内部调用）
- 无新增外部接口暴露
- 无敏感数据回显变更

## v0.1.0 (2026-03-16)

### 核心功能
- FIT 文件解析（基于 fitparse 库）
- 训练指标计算：NP（标准化功率）、TSS（训练压力分数）、IF（强度因子）
- 体能追踪：CTL（慢性训练负荷）、ATL（急性训练负荷）、TSB（训练压力平衡）
- 功率区间与心率区间分布计算
- 功率持续时间曲线 (PDC) 与最佳功率记录

### 高级指标
- eFTP（估算功能阈值功率）
- W'（无氧做功能力）
- xPower（指数加权功率）
- HR drift（心率漂移）
- TRIMP（训练冲量）
- VDOT（跑步能力指数）
- 碳水消耗估算

### 数据同步
- Garmin Connect 活动同步（通过 garminconnect 库）
- FIT 文件自动下载与解析

### API 与服务
- FastAPI REST API（活动查询、分析、体能趋势、PDC、校验）
- SQLite 本地存储（WAL 模式，支持并发读取）

### 工具
- CLI 命令行工具（init、sync、activities、fitness、validate、serve）
- 安装脚本 (`scripts/install.sh`)
